# Changelog

## 1.8.1

- Подвійний клік по будь-якій червоній точці переносить основну систему так, щоб центр гіпотенузи збігся із зеленим плюсом.
- Біля обох червоних точок додано компактні замки із зовнішнього боку основного кута. Одночасно активним може бути один червоний замок.
- При закритих синьому та червоному замках глобальне обертання блокується; протилежну червону точку можна переміщувати для зміни створу без повороту зафіксованого базисного променя.
- При закритому червоному та відкритому синьому замку синя вершина і друга червона точка обертаються навколо зафіксованої червоної точки зі збереженням геометрії кута.
- Стан червоного замка зберігається у налаштуваннях.

## 1.7.0

- подвійний клік по зеленому плюсу переносить його в центр червоної гіпотенузи;
- радіус зелених допоміжних дуг прив’язано до 50% довжини лінії синя вершина → плюс;
- при виході плюса за гіпотенузу додаються червоні пунктирні продовження бокових променів;
- якщо плюс не доходить до гіпотенузи, його промінь продовжується зеленим штрихпунктиром до перетину;
- при виході плюса за межі основного кута дуги стають червоними та з’являється блок `Δ` з кутом виходу;
- пункт меню `+ градуси` перейменовано на `Градуси +`.

## 1.6.0

- додано режим контекстного меню `Дистанція +` із галочкою активного стану;
- поточна лінія від синьої вершини до зеленого плюса автоматично калібрується як `1000м`;
- додано блоки дистанцій для базової лінії, двох червоних променів, гіпотенузи та ліній `Фронт +`;
- подвійний клік по блоку відкриває вбудоване числове редагування;
- підтримано `Enter`, `Esc`, `Backspace`, крапку та кому;
- зміна базової дистанції перекалібровує всі значення;
- зміна дистанції червоного променя синхронно перебудовує обидва промені;
- зміна гіпотенузи перебудовує радіус червоної системи за поточним кутом;
- зміна дистанції `Фронт +` перебудовує червону систему вздовж відповідного променя;
- стан режиму та масштаб метрів зберігаються у файлі налаштувань версії 4;
- додано окремий модуль `src/distance.rs`.
